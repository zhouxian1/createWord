# 离线部署文档

本文档说明 438C 文档生成与知识图谱系统在内网或离线环境中的部署方式，重点覆盖当前已验证的 Ollama `qwen2.5:7b` 模型接入方式。

## 1. 部署组件

| 组件 | 说明 | 是否必需 |
| --- | --- | --- |
| backend | Flask API 服务 | 必需 |
| frontend | Vue 前端 | 必需 |
| Ollama | 本地大模型服务，当前使用 `qwen2.5:7b` | 必需 |
| SQLite / PostgreSQL | 业务数据库，开发可用 SQLite | 必需 |
| ChromaDB | 向量数据库，可使用本地持久化 | 建议 |
| MinIO | 对象存储，不可用时可回退本地文件 | 可选 |
| Neo4j | 图数据库，不可用时可回退本地 JSON 图谱 | 可选 |
| Redis / Celery | 异步任务和队列 | 可选 |

## 2. 推荐端口

| 服务 | 端口 |
| --- | --- |
| 前端 Vite 开发服务 | `3000` |
| 后端 Flask API | `5000` |
| Ollama | `11434` |
| MinIO API | `9000` |
| MinIO Console | `9001` |
| Neo4j Browser | `7474` |
| Neo4j Bolt | `7687` |

## 3. 当前模型部署

当前后端默认调用：

```text
http://192.168.31.245:11434/api/generate
model: qwen2.5:7b
stream: false
```

Ollama 原生验证命令：

```bash
curl http://192.168.31.245:11434/api/generate -d '{
  "model": "qwen2.5:7b",
  "prompt": "请用一句话说明模型已正常运行",
  "stream": false
}'
```

后端建议环境变量：

```env
LLM_PROVIDER=ollama
OLLAMA_API_BASE=http://192.168.31.245:11434
OLLAMA_MODEL=qwen2.5:7b
```

注意：

- Ollama 原生接口不是 OpenAI 兼容 `/v1/chat/completions`。
- 本项目已在 `llm_service.py` 中适配 `/api/generate`。
- 如机器上存在旧的 `LLM_API_BASE=https://dashscope...`，不会影响 Ollama 分支；Ollama 分支读取 `OLLAMA_API_BASE`。

## 4. 离线准备

### 4.1 Python 依赖

在联网环境下载：

```bash
mkdir -p offline_packages/pip
pip download -r backend/requirements.txt -d offline_packages/pip
```

离线环境安装：

```bash
cd backend
python -m venv .venv
.venv\Scripts\Activate.ps1
pip install --no-index --find-links=../offline_packages/pip -r requirements.txt
```

Linux：

```bash
cd backend
python3 -m venv .venv
source .venv/bin/activate
pip install --no-index --find-links=../offline_packages/pip -r requirements.txt
```

### 4.2 Node 依赖

推荐直接打包已安装好的 `node_modules`，或在离线 npm 缓存环境中执行：

```bash
cd frontend
npm install --offline
npm run build
```

开发调试可运行：

```bash
npm run dev -- --host 0.0.0.0
```

### 4.3 Ollama 模型

联网机器：

```bash
ollama pull qwen2.5:7b
ollama list
```

离线迁移时，按 Ollama 实际安装目录迁移模型数据。Windows 常见位置在用户目录下的 `.ollama`，Linux 常见位置在 `/usr/share/ollama/.ollama` 或服务用户目录。迁移后执行：

```bash
ollama list
ollama serve
```

## 5. 后端部署

### 5.1 配置环境变量

可在系统环境变量、`.env` 或启动脚本中配置：

```env
FLASK_ENV=development
STORAGE_TYPE=local

LLM_PROVIDER=ollama
OLLAMA_API_BASE=http://192.168.31.245:11434
OLLAMA_MODEL=qwen2.5:7b

EMBEDDING_MODEL=shibing624/text2vec-base-chinese

MINIO_ENDPOINT=localhost:9000
MINIO_ACCESS_KEY=minioadmin
MINIO_SECRET_KEY=minioadmin
MINIO_BUCKET=doc-gen-system

NEO4J_URI=bolt://localhost:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=neo4j123456
```

### 5.2 启动后端

```bash
cd backend
python run.py
```

Windows 后台启动示例：

```powershell
Start-Process -FilePath python `
  -ArgumentList "run.py" `
  -WorkingDirectory "F:\geovis\cerateWord\backend" `
  -WindowStyle Hidden
```

### 5.3 验证后端

```bash
curl http://localhost:5000/api/project/projects
curl http://localhost:5000/api/438c/document-types
```

验证模型链路：

```bash
curl -X POST http://localhost:5000/api/question/ask-438c \
  -H "Content-Type: application/json" \
  -d "{\"question\":\"请用一句话说明438C是什么\"}"
```

返回中 `status` 应为 `success`。

## 6. 前端部署

### 6.1 开发模式

```bash
cd frontend
npm run dev -- --host 0.0.0.0
```

访问：

```text
http://localhost:3000
```

### 6.2 生产构建

```bash
cd frontend
npm run build
```

构建产物在：

```text
frontend/dist
```

### 6.3 Nginx 示例

```nginx
server {
    listen 80;
    server_name _;

    location / {
        root /opt/doc-gen-system/frontend/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_read_timeout 300s;
        proxy_send_timeout 300s;
        client_max_body_size 500M;
    }
}
```

## 7. 可选组件

### 7.1 MinIO

```bash
docker run -d \
  --name minio \
  -p 9000:9000 \
  -p 9001:9001 \
  -e MINIO_ROOT_USER=minioadmin \
  -e MINIO_ROOT_PASSWORD=minioadmin \
  -v /data/minio:/data \
  minio/minio server /data --console-address ":9001"
```

如果 MinIO 不可达，系统会回退到 `backend/data/uploads` 本地目录。

### 7.2 Neo4j

```bash
docker run -d \
  --name neo4j \
  -p 7474:7474 \
  -p 7687:7687 \
  -e NEO4J_AUTH=neo4j/neo4j123456 \
  -v /data/neo4j:/data \
  neo4j:5-community
```

如果 Neo4j 不可达，知识图谱仍可使用本地 JSON 数据。

## 9. 常见问题

### 9.1 后端仍然请求 DashScope

检查是否在运行旧代码或旧进程。重启后端，并确认：

```bash
python -c "from app.services.llm_service import create_llm_service; s=create_llm_service(); print(s.provider, s.model, s._get_ollama_generate_url())"
```

应输出类似：

```text
ollama qwen2.5:7b http://192.168.31.245:11434/api/generate
```

### 9.2 问答返回很慢

`qwen2.5:7b` 在 CPU 或低性能 GPU 上可能较慢。建议：

- 控制 prompt 上下文长度。
- 优先使用 438C 专项问答优化后的接口。
- 增加 GPU 推理资源。

### 9.3 PowerShell 中文变成问号

这是请求体编码问题。建议使用前端页面测试，或确保 JSON 请求体以 UTF-8 发送。

### 9.4 MinIO 不可用

系统会自动回退到本地文件存储。检查：

```text
MINIO_ENDPOINT
MINIO_ACCESS_KEY
MINIO_SECRET_KEY
MINIO_BUCKET
```

### 9.5 Neo4j 不可用

知识图谱接口仍可返回本地 JSON 图谱。检查：

```text
NEO4J_URI
NEO4J_USER
NEO4J_PASSWORD
```
