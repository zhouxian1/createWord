# 前端界面优化与代码审查

## 优化目标

本次优化面向 438C 文档工程平台的政企和工程管理场景，视觉方向从偏赛博霓虹的展示风格调整为更稳重的高端业务系统风格：浅色工作区、深墨侧栏、金色强调线、多色状态体系和更克制的阴影层级。

## 已完成的界面调整

- 全局主题重构：`frontend/src/styles/theme.css` 统一定义背景、文字、边框、阴影、按钮、表格、表单、标签、弹层等 Element Plus 覆盖样式。
- 首屏工作台优化：`frontend/src/views/Dashboard.vue` 重做了 Hero、指标卡、快速操作、最近文档、文档类型概览的布局和响应式规则。
- 主框架优化：`frontend/src/components/Layout.vue` 调整侧栏、顶部栏、内容区间距和移动端布局，使导航层级更清晰。
- 背景装饰收敛：`frontend/src/App.vue` 移除了大面积模糊光效，避免装饰元素干扰业务信息。

## 视觉规范

- 主背景使用浅灰白工作区，承载表格、表单和卡片时保持高可读性。
- 侧栏使用深墨色，当前菜单用金色左侧标识，不依赖大面积高饱和渐变。
- 组件圆角控制在 8px，卡片、按钮、输入框、弹层保持一致。
- 状态色使用蓝、绿、金、玫红区分动作和结果，避免单一蓝紫色铺满页面。
- 首页卡片和入口按钮使用稳定尺寸，减少 hover 和动态内容导致的布局跳动。

## 代码审查结论

本次审查并已处理的问题：

- `Dashboard.vue` 存在未使用的 `useAppStore` 引入和 `store` 变量，已移除。
- `vite build` 提示主 chunk 超过 500KB，已在 `vite.config.js` 中按 `vue`、`element`、`charts` 拆分 vendor chunk，并为当前全量 Element Plus 引入设置合理告警阈值。
- 原主题依赖外部 Google Fonts，离线部署或内网环境下存在字体加载不稳定风险，已改为系统字体栈。
- 全局背景存在模糊光效装饰，容易与业务系统的审阅、填报、验证场景冲突，已移除。

仍建议后续继续处理：

- 部分业务页面仍保留早期的霓虹字体命名和局部样式，应逐页迁移到全局主题变量。
- 当前缺少 ESLint/Prettier 配置，建议增加格式化和基础静态检查，避免未使用变量进入主分支。
- 当前 `main.js` 全量注册 Element Plus 和图标，后续如需继续压缩首屏资源，可引入按需导入方案。
- `dist/` 已纳入仓库时，每次构建会产生大量产物变更；如果部署流程允许，建议改为 CI 构建产物。

## 验证方式

在 `frontend/` 目录执行：

```bash
cmd /c npm run build
```

PowerShell 环境可能因执行策略拦截 `npm.ps1`，使用 `cmd /c` 可调用 `npm.cmd` 完成构建。
