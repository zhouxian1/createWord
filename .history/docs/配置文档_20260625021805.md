# 438C文档生成与管理系统 - 配置文档

## 1. 环境变量配置

系统通过 `.env` 文件管理环境变量，位于 `backend/.env`。

### 1.1 基础配置

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| FLASK_ENV | development | 运行环境: development / testing / production |
| SECRET_KEY | - | Flask密钥，生产环境必须修改为随机字符串 |

### 1.2 文件存储配置（MinIO）

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| STORAGE_TYPE | minio | 存储类型: minio / local |
| MINIO_ENDPOINT | localhost:9000 | MinIO服务地址 |
| MINIO_ACCESS_KEY | minioadmin | MinIO访问密钥 |
| MINIO_SECRET_KEY | minioadmin | MinIO秘密密钥 |
| MINIO_BUCKET | doc-gen-system | MinIO存储桶名称 |
| MINIO_SECURE | false | 是否使用HTTPS连接MinIO |

**MinIO本地部署**:
```bash
# Docker部署
docker run -d \
  --name minio \
  -p 9000:9000 \
  -p 9001:9001 \
  -e MINIO_ROOT_USER=minioadmin \
  -e MINIO_ROOT_PASSWORD=minioadmin \
  minio/minio server /data --console-address ":9001"
```

**回退到本地存储**:
```env
STORAGE_TYPE=local
```
当MinIO不可用时，系统自动回退到本地文件存储（`data/uploads/`）。

### 1.3 LLM配置 - 国产大模型

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| LLM_PROVIDER | qwen | 模型提供商: qwen / deepseek / chatglm / local |
| LLM_API_BASE | https://dashscope.aliyuncs.com/compatible-mode/v1 | LLM API地址 |
| LLM_API_KEY | - | LLM API密钥 |
| LLM_MODEL | qwen-plus | 使用的模型名称 |

#### 千问(Qwen) - DashScope API

```env
LLM_PROVIDER=qwen
DASHSCOPE_API_KEY=sk-xxxxxxxxxxxxxxxx
LLM_API_BASE=https://dashscope.aliyuncs.com/compatible-mode/v1
LLM_API_KEY=sk-xxxxxxxxxxxxxxxx
LLM_MODEL=qwen-plus
```

可选模型:
| 模型名 | 说明 | 适用场景 |
|--------|------|----------|
| qwen-turbo | 速度快，成本低 | 简单问答、快速生成 |
| qwen-plus | 均衡性价比 | 日常文档生成 |
| qwen-max | 效果最好 | 复杂文档、高质量要求 |
| qwen-long | 超长上下文 | 大篇幅文档生成 |

#### 千问(Qwen) - 本地vLLM部署

LLM_API_BASE=http://localhost:8000/v1

### 1.6 Neo4j 图数据库配置

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| NEO4J_URI | bolt://localhost:7687 | Neo4j Bolt 连接地址 |
| NEO4J_USER | neo4j | Neo4j 用户名 |
| NEO4J_PASSWORD | neo4j123456 | Neo4j 密码 |
| NEO4J_DATABASE | neo4j | 使用的数据库名 |

说明：

- 当前知识图谱采用本地 JSON 与 Neo4j 双持久化
- Neo4j 不可用时，系统会自动回退到本地 JSON，不影响基础功能
- 前端 `D3.js` 图谱页面读取 `/api/knowledge/graph` 接口展示图谱数据

推荐模型:
| 模型 | 说明 |
|------|------|
| shibing624/text2vec-base-chinese | 中文优化，推荐 |
| all-MiniLM-L6-v2 | 英文通用，速度快 |
| paraphrase-multilingual-MiniLM-L12-v2 | 多语言通用 |

## 4. MinIO存储服务配置

### 4.1 存储架构

```
MinIO Bucket: doc-gen-system
├── knowledge/           # 知识库文件
│   ├── {kb_id}/        # 按知识库ID分组
│   │   ├── {uuid}.pdf
│   │   ├── {uuid}.docx
│   │   └── ...
├── 438c/               # 438C标准文档
│   ├── {uuid}.pdf
│   └── ...
├── generated/          # 生成的文档
│   ├── {project_id}/
│   │   ├── {uuid}.docx
│   │   └── ...
└── code/               # 代码文件
    ├── {project_id}/
    │   ├── {uuid}.py
    │   └── ...
```

### 4.2 存储服务特性

- **自动回退**: MinIO不可用时自动回退到本地文件存储
- **预签名URL**: 支持生成临时访问链接（1小时有效期）
- **版本兼容**: 兼容旧数据（本地路径自动转换为MinIO对象名）
