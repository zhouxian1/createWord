# 438C文档生成与管理系统 - 离线部署文档

## 1. 部署架构

```
┌─────────────────────────────────────────────────┐
│                  Nginx 反向代理                    │
│              (端口80/443, SSL终止)                  │
├────────────────────┬────────────────────────────┤
│   前端静态资源       │      后端API服务              │
│   (Vue构建产物)     │   (Gunicorn + Flask)         │
│                    │   (端口5000)                  │
├────────────────────┴────────────────────────────┤
│              数据存储层                             │
│   ┌──────────┬──────────┬──────────┬──────────┐ │
│   │ SQLite/  │ ChromaDB │  Neo4j   │  MinIO   │ │
│   │ PostgreSQL│ 向量库   │ 图数据库 │ 对象存储  │ │
│   └──────────┴──────────┴──────────┴──────────┘ │
├─────────────────────────────────────────────────┤
│              AI服务层（国产大模型）                   │
│   ┌──────────────────┬───────────────────┐       │
│   │  LLM服务          │  Embedding服务     │       │
│   │  千问/DeepSeek/   │ (text2vec-base-   │       │
│   │  ChatGLM(vLLM)   │  chinese)         │       │
│   └──────────────────┴───────────────────┘       │
└─────────────────────────────────────────────────┘
```

## 3. 离线部署准备

### 3.1 在联网环境下载依赖包

#### Python依赖离线包

```bash
mkdir -p offline_packages/pip
pip download -r backend/requirements.txt -d offline_packages/pip
pip download gunicorn -d offline_packages/pip
```

#### Node.js依赖离线包

```bash
cd frontend
mkdir -p ../offline_packages/npm
npm pack --pack-destination ../offline_packages/npm
npm install --prefer-offline
tar czf ../offline_packages/npm_cache.tar.gz node_modules
```

#### 模型文件离线下载

```bash
mkdir -p offline_packages/models

# 下载中文Embedding模型
python -c "
from sentence_transformers import SentenceTransformer
model = SentenceTransformer('shibing624/text2vec-base-chinese')

"
```

#### LLM模型离线下载（本地部署用）

```bash
# 千问模型
```

### 4.2 部署MinIO对象存储

#### Docker部署（推荐）

```bash
# 加载镜像
docker load -i offline_packages/minio-image.tar

# 启动MinIO
docker run -d \
  --name minio \
  -p 9000:9000 \
  -p 9001:9001 \
  -e MINIO_ROOT_USER=minioadmin \
  -e MINIO_ROOT_PASSWORD=minioadmin \
  -v /data/minio:/data \
  minio/minio server /data --console-address ":9001"
```

#### 二进制部署

```bash
# 复制二进制
cp offline_packages/minio /usr/local/bin/
chmod +x /usr/local/bin/minio

# 创建数据目录
mkdir -p /data/minio

# 创建systemd服务
cat > /etc/systemd/system/minio.service << 'EOF'
[Unit]
Description=MinIO Object Storage
After=network.target

[Service]
Type=simple
User=root
Environment=MINIO_ROOT_USER=minioadmin
Environment=MINIO_ROOT_PASSWORD=minioadmin
ExecStart=/usr/local/bin/minio server /data/minio --console-address ":9001"
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF

systemctl daemon-reload
systemctl enable minio
systemctl start minio
```

#### 验证MinIO

```bash
# 访问控制台
curl http://localhost:9001
# 默认账号: minioadmin / minioadmin
```

### 4.8 配置Nginx

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态资源
    location / {
        root /opt/doc-gen-system/frontend/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 后端API代理
    location /api/ {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_read_timeout 300s;
        proxy_send_timeout 300s;
        client_max_body_size 500M;
    }

    # MinIO控制台（可选，仅开发/管理用）
    location /minio/ {
        proxy_pass http://127.0.0.1:9001/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 5. Docker Compose一键部署（推荐）

创建 `docker-compose.yml`:

```yaml
version: '3.8'

services:
  neo4j:
    image: neo4j:5.22-community
    ports:
      - "7474:7474"
      - "7687:7687"
    environment:
      NEO4J_AUTH: neo4j/neo4j123456
    volumes:
      - neo4j_data:/data
    restart: always

  minio:
    image: minio/minio
    ports:
      - "9000:9000"
      - "9001:9001"
    environment:
      MINIO_ROOT_USER: minioadmin
      MINIO_ROOT_PASSWORD: minioadmin
    volumes:
      - minio_data:/data
    command: server /data --console-address ":9001"
    restart: always

  backend:
    build: ./backend
    ports:
      - "5000:5000"
    environment:
      FLASK_ENV: production
      STORAGE_TYPE: minio
      MINIO_ENDPOINT: minio:9000
      MINIO_ACCESS_KEY: minioadmin
      MINIO_SECRET_KEY: minioadmin
      MINIO_BUCKET: doc-gen-system
      NEO4J_URI: bolt://neo4j:7687
      NEO4J_USER: neo4j
      NEO4J_PASSWORD: neo4j123456
      NEO4J_DATABASE: neo4j
      LLM_PROVIDER: qwen
      DASHSCOPE_API_KEY: ${DASHSCOPE_API_KEY}
      LLM_API_BASE: https://dashscope.aliyuncs.com/compatible-mode/v1
      LLM_API_KEY: ${DASHSCOPE_API_KEY}
      LLM_MODEL: qwen-plus
      EMBEDDING_MODEL: shibing624/text2vec-base-chinese
    volumes:
      - backend_data:/app/data
    depends_on:
      - neo4j
      - minio
    restart: always

  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
    volumes:
      - ./frontend/dist:/usr/share/nginx/html
      - ./nginx.conf:/etc/nginx/conf.d/default.conf
    depends_on:
      - backend
    restart: always

volumes:
  neo4j_data:
  minio_data:
  backend_data:
```

```bash
# 启动
docker-compose up -d

# 查看日志
docker-compose logs -f backend
```

## 9. 常见问题

### Q1: MinIO连接失败
检查 `MINIO_ENDPOINT` 是否正确，MinIO服务是否已启动。系统会自动回退到本地存储，不影响基本功能。

### Q2: 千问API调用失败
检查 `DASHSCOPE_API_KEY` 是否正确，网络是否能访问 `dashscope.aliyuncs.com`。如在内网，需配置代理或使用本地部署。

### Q3: 本地LLM模型加载失败
确保已安装NVIDIA驱动和CUDA，vLLM能正确识别GPU。使用 `nvidia-smi` 验证。模型文件需完整下载。

### Q4: 向量库初始化失败
确保 `data/vector_db` 目录有写权限，且 `sentence-transformers` 模型已正确下载到本地。

### Q5: OCR识别中文失败
确保已安装 `tesseract-ocr-chi-sim` 中文语言包，且 `tesseract` 命令在PATH中。

### Q6: 文件上传失败
检查MinIO服务是否正常，bucket是否存在（系统会自动创建）。如使用Nginx，检查 `client_max_body_size` 配置。

### Q7: MinIO磁盘空间不足
定期清理旧版本文件，或配置MinIO的生命周期策略自动清理过期对象。
