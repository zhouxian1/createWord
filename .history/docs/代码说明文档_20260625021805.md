# 438C文档生成与管理系统 - 代码说明文档

## 1. 系统概述

本系统是基于GJB 438C军用软件文档编制规范的文档生成与管理系统，采用Python(Flask) + Vue 3前后端分离架构，实现知识库构建、438C标准文档智能生成、质量验证、文档合稿排版及智能问答等核心功能。

## 2. 技术栈

### 后端
- **框架**: Flask 3.0 + Flask-SQLAlchemy + Flask-Migrate
- **数据库**: SQLite（开发）/ MySQL / PostgreSQL（生产）
- **图数据库**: Neo4j（知识图谱持久化）
- **向量数据库**: ChromaDB（语义检索）
- **AI/ML**: sentence-transformers（向量化）、OpenAI GPT-4（内容生成）
- **文档处理**: python-docx、PyPDF2、pdfplumber、openpyxl、python-pptx
- **OCR**: pytesseract + Pillow
- **代码解析**: tree-sitter（AST解析）
- **中文分词**: jieba

### 前端
- **框架**: Vue 3 + Vite 5
- **UI组件**: Element Plus
- **状态管理**: Pinia
- **路由**: Vue Router 4
- **图谱可视化**: D3.js
- **图表**: ECharts + vue-echarts
- **多模交互**: Web Speech API（语音输入）
- **HTTP**: Axios

## 4. 核心模块说明

### 4.1 知识库构建模块

#### 文档解析 (`document_parser.py`)
- 支持格式: PDF、DOCX、DOC、WPS、XLSX、XLS、PPTX、PPT、TXT、MD、HTML、JSON
- 图片OCR: JPG、JPEG、PNG（通过pytesseract）
- 代码解析: C/C++、Python、Java、.NET（通过AST解析器）
- 二进制文件: 基础元数据提取

#### 文本预处理 (`text_preprocessor.py`)
- 去除无关符号（控制字符、BOM、连续空格/换行、页码标记）
- 文本去重（连续行/段落级去重）
- 敏感词过滤（内置军标敏感词库，支持自定义扩展）
- 代码注释标准化（按语言规范统一注释格式）

#### AST代码解析 (`ast_code_parser.py`)
- 基于tree-sitter的AST解析，支持Python、Java、C、C++、C#
- 提取函数、类、接口、方法、变量、常量等实体
- 提取调用关系图和继承关系图
- 按函数/类/模块粒度分块，保留上下文逻辑

#### 军标文档拆分 (`military_doc_chunker.py`)
- 按"章节-段落-要素"三级层级拆分
- 识别标准编号（1.1.1）、中文编号（一、（一））等多种格式
- 提取表格、图、公式、引用、需求语句、列表项等要素
- 保留父级章节标题作为上下文Metadata

#### 语义索引 (`semantic_indexer.py`)
- 智能分块策略：代码文件按函数/类分块，军标文档按结构化拆分，通用文档按固定长度分块
- 向量化：基于sentence-transformers的Embedding模型
- 倒排索引：基于jieba分词的TF-IDF索引，支持军标关键词加权
- Rerank重排序：结合军标关键词、知识图谱、元数据匹配度综合打分
- 溯源定位：搜索结果附带来源文件、章节、实体等溯源信息

#### 知识图谱 (`knowledge_graph.py`)
- 基于438C标准构建"文档-章节-要素-术语"实体与关系
- 建立代码实体与438C文档要素的映射关系
- 本地 JSON 与 Neo4j 双持久化，兼顾部署灵活性与图数据库扩展能力
- 支持实体查询、关系查询、路径查询

### 4.2 文档生成模块

#### LLM服务 (`llm_service.py`)
- 对接OpenAI API（兼容其他OpenAI格式API）
- 为每种438C文档类型独立配置专业化Prompt
- 支持章节级生成，自动注入上下文信息
- 适配军标表达方式，确保术语规范

#### 文档生成 (`document_generator.py`)
- 三种生成模式：一键生成整篇、按章节独立生成、从代码自动抽取生成
- 自动创建438C标准章节结构
- 支持项目技术图表的自动引用
- 生成任务异步管理

### 4.3 质量验证模块

#### 验证服务 (`validation_service.py`)
- 章节完整性验证：检查是否缺失438C规定的必选章节
- 要素合规性验证：检查必填字段、术语规范、格式约束
- 基于JSON Schema的机器可读约束
- 多级评分体系（优秀/良好/合格/不合格）

### 4.4 文档合稿与排版模块

#### 合稿服务 (`document_merger.py`)
- 多文档合并：支持多份独立文档按指定目录结构无缝合并
- GJB排版规范：自动处理标题层级、字体字号、行距
- 页眉页脚衔接：自动处理合并文档的页眉页脚
- 目录生成：自动生成目录域代码
- 导出格式：Word(.docx)、PDF

### 4.5 智能问答模块

#### 问答服务 (`qa_service.py`)
- 基于知识库的语义检索与LLM生成
- 答案溯源定位：标注答案来源文件、章节、实体
- 438C标准专项问答
- 支持代码上下文关联问答
- 前端支持语音输入并转写为文本问题

## 5. 数据库模型

| 模型 | 说明 |
|------|------|
| Project | 项目信息（系统名称、标识、版本等） |
| KnowledgeBase | 知识库（名称、描述、密级、文档数） |
| KnowledgeFile | 知识库文件（含多维度打标、审核状态、版本管理） |
| Document | 438C文档（类型、状态、生成模式、验证结果） |
| DocumentChapter | 文档章节（编号、标题、层级、内容） |
| GenerationTask | 生成任务（状态、进度、错误信息） |
| MergeTask | 合稿任务（文档列表、目录结构、输出格式） |
| ValidationRule | 验证规则（类型、参数、权重） |
| ReviewRecord | 审核记录（目标类型、动作、审核人、意见） |
| FileVersionHistory | 文件版本历史（版本号、内容哈希、变更说明） |
